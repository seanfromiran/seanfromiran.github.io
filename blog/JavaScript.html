Iteration Declaration: Ruby v. Javascript

Iteration is an essential aspect of most, if not all, programming languages. Even though the basic ideas behind looping do not change between languages, we do have variations in syntax which are important to take note of. So let's take a look at how Ruby and JavaScript handle this universal technique.

Beginning with Ruby, we will examine an if statement.

Relative to other languages, Ruby has a very minimal syntax that feels almost like plain English, and iterators are no exception to this. It is important to observe that elsif is the syntax we use for a conditional statement with more than two expressions to check. Now, let's compare this code block to the same statement in JavaScript.

The most striking difference here is the heavy reliance on parantheses, brackets, and semicolons by JavaScript. These elements have much less importance in Ruby. Also, what elsif signified in Ruby is now else if in JavaScript.

This example illustrates the primary differences between JavaScript and Ruby. With Javascript, we generally have to be more explicit in defining our iterators which means that the syntax is denser.
